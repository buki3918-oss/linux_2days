Linux Round 75: C System Calls - open, read, write, close

Purpose:
Learn low-level Linux file I/O using system calls.

Important concepts:

1. System call

A system call is a request from a user-space program to the Linux kernel.

Examples:

open
read
write
close

2. fopen vs open

fopen returns FILE *.

open returns int fd.

FILE * is a C standard library file object.

fd is a Linux file descriptor.

3. File descriptor

A file descriptor is an integer used by the process to refer to an opened file or device.

Common file descriptors:

0 = stdin
1 = stdout
2 = stderr

The first file opened by open() is usually fd 3.

4. open

open("file.txt", O_WRONLY | O_CREAT | O_TRUNC, 0644)

Meaning:

O_WRONLY: open for writing
O_CREAT: create file if it does not exist
O_TRUNC: truncate file if it already exists
0644: file permission

5. write

write(fd, buffer, size)

Writes bytes to a file descriptor.

6. read

read(fd, buffer, size)

Reads bytes from a file descriptor.

Important:
read does not automatically add '\0' for C strings.

So we often write:

buffer[bytes_read] = '\0';

7. close

close(fd)

Closes a file descriptor.

8. stdout and stderr

write(1, message, size)

writes to stdout.

write(2, message, size)

writes to stderr.

9. strace

strace shows system calls used by a program.

Examples:

strace ./write_syscall

strace -e openat,write,close ./write_syscall

10. Why this matters for firmware / embedded Linux

Many Linux device interfaces are exposed as files or file descriptors.

Examples:

/dev/null
/dev/zero
/dev/tty
/dev/ttyUSB0
/dev/i2c-1
/dev/spidev0.0

Programs often use open/read/write/close to interact with device files.

Important commands:

gcc -Wall -Wextra -g write_syscall.c -o write_syscall
./write_syscall
cat syscall_output.txt

gcc -Wall -Wextra -g read_syscall.c -o read_syscall
./read_syscall

gcc -Wall -Wextra -g copy_syscall.c -o copy_syscall
./copy_syscall syscall_output.txt copied.txt
cat copied.txt

gcc -Wall -Wextra -g stdout_write.c -o stdout_write
./stdout_write

gcc -Wall -Wextra -g stderr_write.c -o stderr_write
./stderr_write

strace -e openat,write,close ./write_syscall

gdb ./write_syscall

Key lesson:
open/read/write/close are lower-level than fopen/fscanf/fprintf/fclose. They are fundamental Linux system calls and are important for embedded Linux, device files, driver testing, and firmware-adjacent development.
