Linux Round 47: cron Scheduling Basics

Purpose:
Practice using cron to run shell scripts automatically on a schedule.

What is cron:
cron is a Linux scheduling tool.
It can run commands or scripts automatically at specific times.

Common use cases:

- Run a script every minute
- Generate a daily report
- Back up files every week
- Clean logs every month
- Run monitoring jobs automatically

Files created:

1. hello_cron.sh
   A simple script that appends current time to cron_test.log.

2. cron_test.log
   Output log generated by hello_cron.sh.

3. cron_notes.txt
   Notes for this round.

Important commands:

1. Check cron command:
   which cron
   which crontab

2. Check cron service:
   service cron status

3. Start cron service:
   sudo service cron start

4. Edit crontab:
   crontab -e

5. List crontab:
   crontab -l

6. Run a script every minute:
   * * * * * /home/yichieh/linux_2days/linux_round47/hello_cron.sh

Cron format:

* * * * * command

Meaning:

minute hour day month weekday command

Example:

* * * * * command
Run every minute.

0 9 * * * command
Run every day at 09:00.

0 9 * * 1 command
Run every Monday at 09:00.

Important concepts:

- cron should use absolute paths.
- ./script.sh may fail in cron because cron does not run from the same working directory.
- cron jobs run in a minimal environment.
- Use log files to check whether cron jobs executed successfully.
- Remove test cron jobs after testing.

Key lesson:
cron can turn a manual shell script into an automatic scheduled task.
