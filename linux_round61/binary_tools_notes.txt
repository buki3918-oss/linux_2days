Linux Round 61: objdump, nm, and readelf Basics

Purpose:
Learn how to inspect object files and executables.

Program:
binary_demo.c

Build commands:

gcc -Wall -Wextra -g -c binary_demo.c -o binary_demo.o
gcc -Wall -Wextra -g binary_demo.o -o binary_demo

Important files:

1. binary_demo.c
   C source code.

2. binary_demo.o
   Relocatable object file.

3. binary_demo
   Executable file.

Tools:

1. nm
   Shows symbols in object files or executables.

2. objdump
   Shows disassembly and section information.

3. readelf
   Shows ELF headers, sections, and symbol tables.

Important nm symbol types:

T
Function or code in the text section.

D
Initialized global variable in the data section.

B
Uninitialized global variable in the bss section.

U
Undefined symbol. It is needed but defined elsewhere.

Important objdump commands:

objdump -d binary_demo.o
Disassemble object file.

objdump -d binary_demo.o | grep -A20 "<add>"
Show assembly around add.

objdump -h binary_demo.o
Show section headers.

Important readelf commands:

readelf -h binary_demo.o
Show ELF header of object file.

readelf -h binary_demo
Show ELF header of executable.

readelf -S binary_demo.o
Show section headers.

readelf -s binary_demo.o
Show symbol table.

Important sections:

.text
Program code.

.data
Initialized global/static variables.

.bss
Uninitialized global/static variables.

.rodata
Read-only data, such as string constants.

.symtab
Symbol table.

.strtab
String table.

.debug_info
Debug information generated by -g.

Key lesson:
nm, objdump, and readelf help inspect what is inside object files and executables. These tools are useful for C, firmware, embedded Linux, and low-level debugging.
